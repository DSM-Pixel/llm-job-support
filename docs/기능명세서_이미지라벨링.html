제품명 자연어 기반 이미지 이해 · 자동 라벨링 도구
대상 과제 도로 파손(포트홀/균열) 라벨링 자동화 등 Vision AI 데이터 구축
사용자 라벨링 작업자 · 데이터 검수자 (비개발 직군 포함)
플랫폼 웹 (Gradio 기반, 단일 화면 + 탭 구조)
버전 v0.4 프로토타입 / 작성일 2026-06-19
문서 목적 현재 동작 중인 프로토타입의 기능·화면·상태를 디자이너에게 전달하여 UI/UX 리디자인 기준선 제공

1. 제품 한 줄 정의

사용자가 이미지를 올리고 "포트홀을 찾아줘"처럼 찾을 대상을 한국어로 입력하면, AI가 해당 객체의 바운딩 박스를 자동으로 그려주고, 사용자가 이를 검수·수정한 뒤 학습용 데이터셋(YOLO/COCO) 형식으로 내보내기/저장하는 도구.

핵심 가치: "사람이 박스를 일일이 그리던 작업"을 AI가 1차 초안으로 만들고, 사람은 확인·보정만 하도록 바꿔 라벨링 속도를 끌어올린다. → 디자인은 AI 결과 검수 흐름이 가장 중요.

2. 전체 화면 구조 (정보 구조)

상단 단일 페이지 + 작업 단계별 탭으로 구성. 모든 탭은 공통적으로 좌측 이미지 캔버스 / 우측 컨트롤 패턴을 따른다.

#탭(기능)한 줄 역할핵심 산출
1📝 설명 분석이미지를 글로 이해 (설명/목록/이상탐지)텍스트 리포트
2🔲 박스로 찾기 핵심대상을 자동 탐지해 박스 그림라벨(박스) 세트
3✏️ 수동 보정AI 박스를 표/마우스로 수정수정된 라벨
4🖱️ 마우스 라벨링AI 없이 직접 드래그로 박스 작성수기 라벨
5📁 배치 처리여러 장 한 번에 자동 라벨링일괄 라벨 + 진행률
6💾 내보내기 / 저장YOLO·COCO 변환, 백엔드 저장다운로드 파일
7📊 통계 대시보드저장된 라벨 집계 시각화차트/표

3. 화면별 상세 명세

화면 ① 설명 분석

흐름: 이미지 업로드 → 시나리오 프리셋 선택(또는 직접 질문) → [분석 실행] → 결과 텍스트가 실시간(스트리밍)으로 흘러나옴
입력이미지 1장, 프리셋 드롭다운(4종) 또는 자유 입력 텍스트박스
프리셋도로 파손/포트홀 찾기 · 이미지 전체 설명 · 객체 목록 뽑기 · 이상 상황 탐지
출력한국어 분석 결과(마크다운 텍스트). 글자가 타이핑되듯 채워짐
디자인 포인트스트리밍 중 "생각하는 중" 로딩 표현 필요. 직접 입력 시 프리셋은 무시됨을 시각적으로 안내

화면 ② 박스로 찾기 가장 중요

흐름: 이미지 업로드 → 찾을 대상 입력(쉼표로 여러 개) → 엔진 선택 → 신뢰도 임계값 설정 → [찾기] → 캔버스에 박스 + 라벨 오버레이 표시
대상 입력예: 포트홀 또는 포트홀, 균열, 맨홀 (다중 클래스)
엔진 선택① Gemini(자연어 이해, 유연) / ② YOLO-World(박스 위치 정확) — 라디오/토글
신뢰도 필터0~100% 슬라이더. 임계값 미만 박스는 자동 숨김
결과 표시이미지 위 컬러 박스 + 클래스명 + 신뢰도(%). 클래스별 색상 자동 배정(10색 팔레트)
결과 요약"N개 탐지 / M개 필터로 숨김" 형태 상태 메시지
┌─────────────────────────────┐ ┌───── 컨트롤 ──────┐ │ │ │ 찾을 대상 │ │ [이미지 + 박스 오버레이] │ │ [포트홀, 균열 ] │ │ ┌──────┐ │ │ 엔진 ◉Gemini ○YOLO│ │ │포트홀 │92% │ │ 신뢰도 ──●──── 40%│ │ └──────┘ ┌────┐ │ │ [ 찾 기 ] │ │ │균열│78% │ │ ▸ 3개 탐지, 1개 숨김│ └─────────────────────────────┘ └───────────────────┘
주의: AI 박스는 초안이다. 사용자가 "이거 틀렸네" 하고 바로 고칠 수 있어야 하므로, ②번 결과에서 ③ 수동 보정으로 자연스럽게 이어지는 동선이 핵심. (현재는 같은 탭 하단에 보정 영역이 붙어 있음)

화면 ③ 수동 보정

흐름: ②의 탐지 결과를 표(스프레드시트)로 표시 → 좌표·클래스명 직접 수정 / 행 추가·삭제 → [반영] → 캔버스 다시 그림
편집 방식표 형태(클래스명, 좌표 4값, 신뢰도). 셀 직접 편집 + 행 추가/삭제
정밀 마스크[정밀화] 버튼 — 박스를 픽셀 단위 외곽선(폴리곤)으로 변환 선택
디자인 포인트표 편집은 비개발자에게 어려움 → 마우스로 박스를 끌어 수정하는 직접조작 UI가 이상적(개선 여지). 좌표는 0~1000 정규화 정수

화면 ④ 마우스 라벨링

흐름: 이미지 업로드 → 마우스로 영역 드래그 → 클래스 선택 → [저장]
기본 클래스포트홀 · 균열 · 맨홀 · 차량 · 사람 · 표지판 (확장 가능)
용도AI가 못 찾는 객체를 사람이 직접. 순수 수동 라벨링 지원

화면 ⑤ 배치 처리

흐름: 여러 이미지 업로드 → 찾을 대상 입력 → [일괄 실행] → 진행률 바 + 장별 결과 누적
진행 표시"3 / 12 장 처리 중…" 진행률. 한 장 실패해도 멈추지 않고 계속
옵션SAM 정밀 마스크 적용 여부, 호출 간 딜레이(초) 조절(한도 완화)
디자인 포인트장시간 작업 → 진행률·예상시간·실패 장 표시가 중요. 결과 썸네일 그리드 권장

화면 ⑥ 내보내기 / 저장

포맷YOLO(.txt) · YOLO-seg(폴리곤 포함) · COCO(.json)
다운로드포맷별 zip 또는 개별 파일
백엔드 저장[서버에 저장] 버튼 현재 MOCK — 추후 실제 저장소 연동 예정
데이터셋저장 라벨을 학습용 train/val로 분할 내보내기(검증 비율 슬라이더)

화면 ⑦ 통계 대시보드

집계클래스별 라벨 수(막대), 신뢰도 분포, 이미지별 박스 수(표)
용도데이터셋 균형·품질 점검. 어떤 클래스가 부족한지 한눈에

4. 공통 컴포넌트 / 상태 정의

상태표현 요구
빈 상태(이미지 없음)"이미지를 먼저 업로드해주세요" 안내. 실행 버튼 비활성
로딩/처리 중스트리밍 텍스트 또는 진행률. 버튼 중복 클릭 방지
탐지 결과 없음"해당 대상을 찾지 못했습니다" — 오류 아님, 정상 안내
API 한도 초과(429)"무료 사용 한도 초과 — 약 10~30초 뒤 다시 시도하세요" (무료티어 특성, 자주 발생)
키 미설정탐지 기능 사용 시 키 필요 안내. 상단 전역 배너 권장(개선)
박스 오버레이클래스별 10색 팔레트, 라벨 칩(클래스명+신뢰도%), 마스크는 반투명 채움
색1 색2 색3 색4 색5 … 클래스별로 순환 배정(총 10색)

5. 참고 — 데이터 규약 (디자인엔 영향 적음)

  • 좌표: [ymin, xmin, ymax, xmax], 0~1000 정규화 정수 (모든 화면 공통)
  • 신뢰도(confidence): 0~100 정수
  • 라벨 1건 = { 클래스명, 박스좌표, (선택)신뢰도, (선택)폴리곤 }

6. 디자인 우선순위 제안

  1. ②→③ 검수 동선 — AI 박스 확인·수정이 가장 잦은 작업. 직접조작(드래그 수정) UX 최우선.
  2. 상태 피드백 — 한도초과/로딩/빈결과 메시지를 명확하고 비위협적으로.
  3. 배치 진행률 — 장시간 작업의 신뢰감.
  4. 비개발자 친화 — 표 편집 의존을 줄이고 시각적 조작 위주로.
